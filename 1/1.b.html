전달인자(argument) 함수로 전달되는 값.

대입(assignment) 객체의 현재 값을 지우고 새로운 값으로 대체하는 것.

블록(block) 중괄호로 묶인 0개 이상의 일련의 문장들.

버퍼(buffer) 데이터를 붙잡기 위한 저장 구역. IO 기능은 입력을 (또는 출력을) 버퍼에 저장하고 프로그램상의 행동과 독립적으로 버퍼에 읽고 쓴다. 출력 버퍼는 버퍼가 쓰일 수 있도록 강제로 비워질 수 있다. 디폴트로 cin을 읽는 것은 cout을 비운다. cout은 프로그램이 정상적으로 끝날 때도 비워진다.

내장형 타입(built-in type) 언어에 의해 정의된 타입으로, 가령 int.

cerr 표준 오류와 결부된 ostream 객체다. 종종 표준 출력과 같은 장치에 쓴다. 디폴트로, cerr에 쓰는 일은 버퍼와 무관하다. 주로 오류 메시지나 프로그램의 정상적인 논리에 부합하지 않는 출력에 쓰인다.

문자 스트링 리터럴(character string literal) 문자열 리터럴의 다른 표현

cin 표준 입력에서 읽어 들이기 위한 istream 객체.

클래스(class) 연관 연산과 함께 우리 자신만의 자료 구조를 정의하기 위한 기능. 클래스는 C++의 가장 근본적인 기능 가운데 하나다. istream이나 ostream 같은 라이브러리 타입이 바로 클래스다.

클래스 타입(class type) 클래스에 의해 정의되는 타입이다. 타입의 이름은 클래스의 이름이다.

clog 표준 오류에 결부된 ostream 객체다. 디폴트로, clog에 쓰는 일은 버퍼와 연관된다. 주로 로그 파일에 프로그램 실행에 관한 정보를 보고하기 위해 쓰인다.

주석(comments) 컴파일러가 무시하는 프로그램 텍스트다. C++에는 두 가지 주석이 있다. 단일 라인 주석과 쌍 주석이 그것이다. 단일 라인 주석은 //로 시작한다. //에서 라인 끝까지 전부 주석이다. 쌍 주석은 /*로 시작해 */까지 전부 주석으로 처리한다.

조건(condition) 참 또는 거짓으로 평가되는 표현식이다. 0의 값은 거짓이며 그 외 값은 전부 참이다.

cout 표준 출력으로 쓰기 위한 ostream 객체다. 보통 프로그램의 출력을 작성하는 데 쓰인다.

중괄호(curly brace) 블록을 경계 짓는다. 여는 중괄호({)는 블록을 시작하고, 닫는 중괄호(})는 블록을 끝낸다.

자료 구조(data structure) 데이터와 그 데이터상의 연산의 논리적인 구획.

편집-컴파일-디버그(edit-compile-debug) 프로그램이 제대로 실행하게 만들기 위한 과정.

end-of-file 시스템 특정적인 표지로, 파일상에 더는 입력이 없다고 암시.

표현식(expression) 계산의 가장 작은 단위. 표현식은 하나 이상의 피연산자와 주로 하나 이상의 연산자로 구성된다. 표현식은 결과를 생산하는 것으로 평가된다. 가령 i와 j가 int라면, i+j는 표현식이며 두 int값의 합을 취한다.

for 문장 반복 실행을 가능케 하는 반복문. 주로 계산을 고정된 횟수만큼 반복하는 데 쓰인다.

함수(function) 계산의 명명된 단위.

함수 몸체(function body) 함수에 의해 수행되는 행동을 정의하는 블록.

함수 이름(function name) 함수가 알려지고 호출될 수 있는 이름.

헤더(header) 클래스의 정의나 다른 이름들이 다양한 프로그램에 쓰일 수 있도록 하는 메커니즘. 프로그램은 #include를 통해 헤더를 사용한다.

if 문장 특정 조건의 값에 기초한 조건적인 실행. 조건이 참이라면 if 몸체가 실행된다. 아닐 경우, 또 else가 있다면 else 몸체가 실행된다.

초기화(initialize) 객체가 생성될 때 그 객체에 값을 부여하는 것.

iostream 스트림-지향적 입력 및 출력을 위한 라이브러리 타입을 제공하는 헤더

istream 스트림-지향적 입력을 제공하는 라이브러리 타입

라이브러리 타입(library type) 표준 라이브러리에 의해 정의되는 타입으로, istream을 예로 들 수 있다.

main C++ 프로그램을 실행하기 위해 운영체제가 호출하는 함수. 각 프로그램은 단 하나의 main 함수를 가져야만 한다.

조작자(manipulator) 읽히거나 쓰일 때 스트림 자체를 “조작하는” 객체로, std::endl을 예로 들 수 있다.

메소드(method) 멤버 함수의 다른 말.

이름공간(namespace) 라이브러리에 의해 정된 이름들을 단일의 장소에 배치하는 메커니즘. 이름공간은 의도하지 않은 이름 충돌을 피하게 돕는다. C++ 라이브러리에 의해 정의된 이름들은 std 이름공간에 있다.

ostream 스트림-지향적 출력을 제공하는 라이브러리 타입.

매개변수 목록(parameter list) 함수 정의의 일부분. 빈 리스트일 수 있음. 어떤 인자들이 함수 호출에 쓰일 수 있는지 특정.

반환 타입(return type) 함수에 의해 반환되는 값의 타입

소스 파일(source file) C++ 프로그램을 포함하는 파일을 묘사하기 위한 용어

표준 오류(standard error) 오류 보고를 위해 쓰이는 출력 스트림. 보통 표준 출력과 표준 오류는 프로그램이 실행되는 창에 결부된다.

표준 입력(standard input) 주로 프로그램이 실행되는 창에 결부된 입력 스트림.

표준 라이브러리(standard library) 모든 C++ 컴파일러가 지원해야 하는 타입과 함수의 모음. 라이브러리는 입출력을 지원하는 타입을 제공한다. C++ 프로그래머는 단순하게 “라이브러리”라고 표준 라이브러리 전체를 지칭한다. 또 라이브러리의 특정 부분을 언급할 때 가령 “iostream 라이브러리”라며 라이브러리 타입을 언급하는 경향을 보인다.

표준 출력(standard output) 주로 프로그램이 실행되는 창에 결부된 출력 스트림.

문장(statement) 프로그램이 실행될 때 취하는 행동을 특정하는 프로그램의 일부분. 세미콜론이 뒤따르는 표현식이 곧 문장. 다른 종류의 문장에는 블록과 if, for, while이 포함되며, 이들 모두 다른 문장을 자기 자신과 더불어 포함한다.

std 표준 라이브러리에 의해 쓰이는 이름공간의 이름. std::cout은 우리가 지금 std 이름공간에서 cout이라고 정의된 이름을 쓰고 있다는 것을 암시한다.

문자열 리터럴(string literal) 큰따옴표로 묶인 일련의 0개 이상의 문자.

초기화되지 않은 변수(uninitialized variable) 초깃값이 주어지지 않은 변수. 초깃값이 없는 클래스 타입의 변수는 클래스 정의에 따라 초기화된다. 함수 내부에서 정의된 내장형 타입의 변수는 초기화하지 않는 이상 초기화되지 않는다. 초기화되지 않은 변수를 사용하는 것은 오류로 이어진다. 초기화되지 않은 오류는 풍부한 버그의 원천이다.

변수(variable) 명명된 객체.

while 문장 어떤 조건이 참인 한 반복 실행을 제공하는 반복문. 몸체는 조건의 진리값에 따라 0번 이상 실행된다.

() 연산자 호출 연산자. 함수 이름에 뒤따르는 “()”. 이 연산자는 함수 실행을 유도한다. 괄호 안에 인자가 전달될 수도 있다.

++ 연산자 증가 연산자. 피연산자에 1을 더한다. ++i는 i = i + 1과 동치다.

+= 연산자 복합 대입 연산자로 오른쪽 피연산자를 왼쪽 피연산자에 더한 다음 그 결과를 왼쪽 피연산자에 저장한다. a += b는 a = a + b와 동치다.

. 연산자 점 연산자. 좌측 피연산자는 반드시 클래스 타입의 객체여야 하며 우측 피연산자는 반드시 그 객체의 멤버의 이름이어야 한다. 연산자는 주어진 객체의 명명된 이름을 취한다.

:: 연산자 범위 연산자. 다른 용례와 더불어, 범위 연산자는 이름공간 접근 목적으로 쓰인다. 가령 std::cout은 std 이름공간에서 cout이라는 이름을 취한다는 것을 뜻한다.

= 연산자 우측 피연산자의 값을 좌측 피연산자로 표기되는 객체에 대입한다.

--연산자 피연산자에서 1을 뺀다. --i는 i = i - 1과 동치다.

<< 연산자 출력 연산자. 우측 피연산자를 좌측 피연산자에 의해 암시되는 출력 스트림에 쓴다. cout << “hi”는 hi를 표준 출력에 쓴다. 출력 연산자는 묶일 수도 있다. cout << “hi” << “bye”는 hibye를 쓴다.

>> 연산자 입력 연산자. 좌측 피연산자에 의해 특정되는 출력 스트림에서 우측 피연산자로 값을 읽어 들인다. cin >> i는 표준 입력상의 다음 값을 i로 읽어 들인다. 입력 연산자는 묶일 수도 있다. cin >> i >> j는 우선 i로 읽어 들인 다음 j로 읽어 들인다.

#inlcude 코드상의 헤더를 프로그램에서 쓸 수 있도록 만드는 지시사.

== 연산자 상등성 연산자. 좌측 피연산자가 피연산자와 상등인지 검사한다.

!= 연산자 불일치 연산자. 좌측 피연산자가 피연산자와 상등이 아닌지 검사한다.

<= 연산자 이하 연산자. 좌측 피연산자가 우측 피연산자보다 작거나 같은지 검사한다.

< 연산자 미만 연산자. 좌측 피연산자가 우측 피연산자보다 작은지 검사한다.

>= 연산자 이상 연산자. 좌측 피연산자가 우측 피연산자보다 크거나 같은지 검사한다.

> 연산자 초과 연산자. 좌측 피연산자가 우측 피연산자보다 큰지 검사한다.
